Corner Detection

Corner detection is used to solve the problem of image stitching, which is when we have multiple images that we wish to transform into one big panorama.

Why not use edge detection? Along an edge, all the points have the same neighbors.

If this image represents an edge, then you will find that along it, all the points have the same neighbors, so how can we extract features from this? Therefore, it is not suitable.

So what makes a point interesting for us? For a mountain, it could be its mountaintop.

The points along the edges of the mountain all look the same with the same neighbors. However, at the corners (here the mountain top and where it levels off with the ground), are decidedly different from the rest of the mountains. We can then consider them feature points.

What made us consider these points?

  1. They must exist. Any mountain must have a mountaintop.
  1. They are different from any other point in the image.

In feature detection ‚Üí we want distinct and repeatable features, so when we look at other images, we get the same features for the same object.

Our pipeline for image stitching has 3 steps

  1. Detection
  1. Description
  1. Matching

The description step is done mathematically, and we use it to deduce the transformation between 2 images, which allows us to successfully warp one image and stitch it into another and put them into one space/plane.

Applications on detecting corners

  • Image alignment (homography, fundamental matrix)
  • 3D reconstruction
  • Motion tracking
  • Object recognition
  • Indexing and database retrieval
  • Robot navigation: obstacle avoidance by robot localization. If a robot wants to make it across a room without bumping into anything, the first step is feature detection and then optical flow. If a part of the image has more optical flow, meaning more moving objects, the robot then changes course and moves in another path.

Instance recognition

A database of planar objects is formed during training, and then these are used in instance recognition during tests. The objects may then have a different orientation, or be occluded (part of it hidden by other objects).

The same can be done for 3D objects

Location Recognition

To find out what a location is, local features can be use to identify which location this is.

Image matching

Sometimes images undergo severe viewpoint change (perspective).

We have two tasks

  • Detect features that you are sure will be found in other images
  • When found, describe those features such that matching is accurate

Computer vision can detect features that are very hard to see with the naked eye

Feature selection

Which points in the image are more useful for us, the red or yellow? We want points that will help us find the object in other images, even when it looks different.

All yellow points are equally similar, as they lie on the edge. The points of interest would be the ones that lie on the corners. That‚Äôs why corners are a popular feature in object detection.

Feature Descriptors

What features do we want? The process of choosing features, called feature engineering, is a popular problem that relies on intuition and vague empirical notions. Now with deep learning techniques, end-to-end solutions exist that involve feature extraction learning, so that we can do without the feature engineering step.

Why do we need feature descriptors? If we have extracted good features, then this is just the start of the pipeline. Now we need to actually match them, so if we know where the good features are, how do we match them?

Designing feature descriptors

There are many kinds of transformations that we need to take into account

Photometric transformations

They represent different lighting conditions

Geometric transformations

Objects will appear at different scales, translation, and rotation.

Our feature descriptors need to be invariant to both, and find matches regardless of orientation or lighting. The descriptors will be able to recover and find out what geometric transformation was applied.

What is the best descriptor for an image feature? Let‚Äôs explore some possibilities:

  1. Position and scale

    Possibly the worst feature descriptor; it will almost definitely change. A feature descriptor must describe the patch and get a vector that will be a signature for the patch.

  1. Image patch

    If the geometry and appearance change, this will not be useful as a feature descriptor.

  1. Image gradients

    This makes it invariant to absolute intensity changes. However, any deformations will make this a bad feature descriptor.

  1. Color histogram

    While this is invariant to changes in scale and rotation, you can easily fool the descriptor as 2 very different objects can have the same histogram. Spatial information encoding is needed.

    1. Spatial histogram

    Getting the color histogram for patches in the image. It is still not invariant to rotation. A solution would be to find out the dominant orientation, and rotate the image towards it. Finding the derivative proves to be more robust.

We will apply all of these ideas in developing sophisticated descriptors.

  • Histograms
  • Dominant orientation - orientation normalization
  • Derivatives

Some descriptors provide a holistic presentation, describing the entire image, while others describe a small part, called patches, of an image.

GIST descriptor

For each patch of an image, 50 filter responses are calculated, each on a different filter where we look for a specific feature. For example one could be the derivative, one could depend on the scale, and so on. We use a kind of ensemble voting system, where we then compute the average of these filters over a small patch.

HOG descriptor

Histogram oriented gradient descriptor.

For each block of cells in an image, we will find a histogram of gradient magnitudes, where the x-axis will have some values for the angles, and the y-axis will represent the gradient magnitude of the angle bin. So for example, if within the cell 4 pixels have a gradient of 45 degrees, each having a magnitude of 5, 4, 7, and 10 respectively, then the 45 degrees bin will have a value of 5+4+7+10 = 26.

Afterwards the cell histograms will be concatenated with the other cell histograms within the same block, giving us our descriptor for that block.

SIFT

Scale Invariant Feature Transform

It is a local feature descriptor. It describes both a detector and a descriptor.

The detector is very robust and reliable, and the descriptor extracts signature from blobs and is insensitive to things like rotation and lighting, as it relies on rescaling and reorienting to match the template and the lighting also has less impact.

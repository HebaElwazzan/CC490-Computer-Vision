Non-Linear Filtering

What are the perks of linear filtering?

  • We can make use of convolution‚Äôs properties
  • The computational complexity is low
  • Separable filters

Sensors introduce noise and defects. The defects can be things such as salt and pepper noise where some pixel values disappear, or additive gaussian noise, which changes the intensity values of the pixels and may introduce artifacts that may blur the image where the effect is unwanted.

Median Filter

This is a kind of non-linear filter where the values in the neighborhood (that the kernel covers, so if a kernel is a 3x3, the neighborhood comprises of 9 pixels) are sorted and replacing spikes with the median value.

The missing values in the salt and pepper noisy image are outliers. These can be replaced by the local median using the median filter to smooth the image out.

The good thing about the median filter is that it is edge preserving. Meaning that if there is an edge, it will not be considered an outlier, and the filter will not attempt to replace. How is this achieved? The filter takes values from the neighborhood, so if the neighborhood indicate there is an edge, it will remain an edge. However, in the mean filter, the mean is computed over the entire image, so the edge is blurred.

What about the complexity? The median filter has a complexity of O(M2nlogn)O(M^2nlogn)O(M2nlogn)Ôªø where M2M^2M2Ôªø is the number of pixels in the image of size MxM. This complexity is worse than the linear filtering where the complexity is O(k2√óM2)O(k^2\times M^2)O(k2√óM2)Ôªø where k is the dimension of the kernel, and n is kxk.

Weighted Mean

The image is multiplied by a weight kernel first that defines which areas are to be given more significance than others. As indicated by the previous figure, the kernel acts as an indicator as to which pixels should be repeated and how many times. For example the pixel in the position upper center, having the value 7, has the respective weight of 2. Therefore it is repeated twice.

The new array of pixel values is then sorted in order to find out the weighted median. In this case it is 2, whereas had we not introduced the weights, it would have been 3.

Bilateral Filtering

What if want to remove Gaussian noise without blurring details of the image? Bilateral filtering smoothens an image with a degree inversely proportional to the local contrast. It is a non-linear filter that is the product of two filters.

What does this mean?

  • When an area is flat, this means the contrast is 0, so we use gaussian blurring
  • If we have an edge, corner, or texture, the contrast here will be higher, so we will use less blurring in this case.

The key is to no smooth blindly and contrast of the area directly affects the degree of smoothing.

The filter depends on both the gaussian filter and the group of pixels around the pixel being filtered.

Boundaries

There is a problem associated with boundaries in images. If the image size is fffÔªø and the kernel size is gggÔªø, then the pixels at the edge of the image will have the problem of not having enough neighbors for the kernel to fit over. So how do we handle this?

MATLAB has 3 options

  • full: output size is sum of sizes of f and g
  • same: output size is same as f
  • valid: output size is difference of sizes of f and g

OpenCV has just one mode: same

But then how do we deal with the filter window falling off the edge of the image.

There are multiple methods:

  • clip filter
  • Wrap around
  • copy edge
  • reflect across edge

We usually use reflect over edge method as it is considered the best one (subjective as that may be).

Hough Transform

Finding boundaries

When would edge detection fail? So far we were assuming that an edge equates a boundary in an image, but that may not always be the case. An edge represents a large discontinuity in intensity, like the difference between foreground and background. In an edge, it is obvious that there is an object change.

However, what if we have an image like this

The two elephants are of the same kind. There is no obvious discontinuity between them at the boundary between them. In this case, edge detection will fail to find a boundary.

What defines the existence of a boundary here? The transition in this case is a discontinuity in depth rather than intensity, reflecting the presence of a boundary.

There is also the definition of boundary. It depends on each task. For example, take the picture below of grains. Do we want to detect the boundary between each grain, or the general shape of the sack?

Another problem with edge detection is that the amplitude is high in general in some areas of the image (darker regions) and low in other regions (lighter ones). This entails that shadows will always appear as an edge.

Multi-scale edge detection

We can conclude that different resolutions need different edge detectors. Do we want the edges of objects and entities, or do we want the details?

Noisy edges

If there are noisy edges in an image, no clear edge will be detected.

The measures that the edge detectors use may not always align to what we think is an edge.

Line Fitting

If edge detection returns a lot of pixels, how do we find an exact line? We will try to group some pixels into one line using line fitting.

We can to minimize the average square distance (E) to find the best fit line that passes through the pixels the edge detector decided lie on an edge. This can be done by taking its derivative, equating it with zero, and finding m and c using a close form solution.

But is this the best way? Any edge detector will return some pixels as feature points, and those features cannot be assumed to be lying on just one line. However, line fitting, by definition, returns just one line. Line fitting would be a good approach only on the condition that we have just one line in the image, which is an abnormal constraint.

Hough space

Every point in the Hough space is a line equation. Which are the edges in the images? The points that are peaks in the Hough transform, ie. the points where there are a lot of intersections ‚Äî the highly bright regions.

How to vote in Hough space?

We can try model fitting. However, there are multiple difficulties

  • Clutter ‚Üí like in the grains picture, where is the boundary?
  • Multiple lines in an image (like the two intersecting lines that represent the edge of a mountaintop) cannot have the same model.
  • Missing data ‚Üí the whole model doesn‚Äôt appear in the picture.
  • Noise will create lines that do not exist. The possible lines are too many since any 2 points can have a line between them.

It is not feasible to check all combinations of features by fitting a model to each possible subset

Image and Parameter Space

Hough transform depends on converting the image space into a parameter space, where the model‚Äôs parameters (for example, m and b for a line model), become the variables in the parameter space. The parameter space is what we call the Hough space.

A point in the image space has a lot of lines passing through it (our possible boundaries). All of these lines are points in the parameter space, thus forming a line.

Do 2 points always represent an intersection in the parameter space? It depends. If the 2 points have a slope of infinity between them (form a vertical line), then in the parameter space, they will form parallel lines.

In this case, where is our edge? Hough transform relies on voting, meaning that the more lines intersect at a point in the Hough space, the more likely that this point represents a line (an edge) in the the image space.

Hough Circles

We will apply the same idea of parametrization, but this time to a circle.

Our variables in the Hough space in this case will be ((a, b), r), where a and b represent the circle‚Äôs center and r represents the radius.

If radius is known:

Image SpaceHough (Parameter) Space
pointcircles
2 points2 intersections
3 points or more1 intersection point

What if radius is unknown? The Hough space dimensionality increases to 3 (becomes 3D) and the surface shape becomes a cone, as we vary the radius and get a different circle each time around the same center, which forms a cone. With the curse of dimensionality, computations become a lot more complicated and lenghthier.

So this got a little bit complicated. How do we make things simpler? We can utilize gradient information.

Rather than having to vote for entire circle around the point (x, y), we can use the gradient to pinpoint a point that lies on the radius of the circle, assuming we know what the radius is. We can easily find the gradient of the point using edge detection.

This changes nothing in the parameter space, but speeds up the computation.
